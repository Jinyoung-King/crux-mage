# 중간 이벤트 시스템 설계 (반복성/엔드게임 1축)

작성 2026-06-15, v1.94~. 목표: "몇 판 하면 매 판 똑같아 지루"의 마지막 조각 = **런 내 다양성·의미 있는 분기**. (엔드게임 다른 축 '상승 계층'은 이후 별도.)

## 뼈대
기존 "웨이브 클리어 → 카드 드래프트" 흐름에서 **특정 웨이브의 드래프트 자리를 이벤트로 교체**. 상점(`_open_shop`)과 동일 구조: 일시정지 → 패널 → 선택 적용 → 다음 웨이브.
- **신규 `scenes/ui/event_select.gd`**(코드 빌드 Control, card_select와 별개·단순): `open(title, flavor, choices)` → `chosen(index)` 시그널. 딤+중앙 패널+제목+분위기글+선택지 버튼 N개.
- main: `_open_event(kind)` 가 kind별 선택지 구성·패널 표시·`chosen`→`_on_event_choice(kind, idx)`. 적용 후 `_start_wave(next)`.
- **온보딩 보호**: 이벤트는 **wave 11+** 부터(첫 10웨이브는 현행 유지).

## 배치 (끝자리, 10주기 — 0보스·3/5/7중간보스·6상점·8보너스와 안 겹침)
- 끝자리2(12·22…) → **제물 제단**
- 끝자리4(14·24…) → **원소 균열**
- 보스 클리어 직후(11·21…) → **갈림길**

## 이벤트 3종
### ① 원소 균열 (v1.94 — 첫 구현, 기존 5속성·반응과 시너지)
5속성 중 1택 → **이번 런 동안 그 속성 스킬 위력 +30%**(중복 균열 누적). 거절 없음(항상 이득).
- 신규 `BuildState.element_empower: Dictionary` {원소:보너스}. `player.eff_power(s)`에서 스킬 속성(`SkillLib.DEFS[s.id].element`) 일치 시 `× (1 + element_empower.get(elem,0))`.
### ② 제물 제단 (v1.95)
2지선다: "기지 내구도(최대체력) −20 → 희귀+ 카드 확정" / "거절". 보상 = `_draw_cards(1, rare=true)[0]` → `apply_card`. 비용 = `max_hp -= 20`(하한 보장).
### ③ 갈림길 (v1.96)
2지선다: "안전: 체력 30% 회복" / "도전: 다음 2웨이브 적 +30%, 클리어 시 전설 확정 카드". 신규: 임시 난이도 배율(2웨이브 카운트다운) + 다음 드래프트 희귀도 바닥(legendary).

## 분할 구현
- **v1.94**: 뼈대(event_select + _open_event 라우팅 + 흐름) + **원소 균열**.
- **v1.95**: 제물 제단.
- **v1.96**: 갈림길.
각 단계 헤드리스 검증(패널 생성·선택 적용·효과 반영·배치·무오류) 후 배포.

## 검증 포인트
- 균열: 선택 후 `element_empower[elem]==0.3`, eff_power가 해당 속성 스킬에 ×1.3.
- 라우팅: wave<11 이벤트 없음, 끝자리4(≥14)에서 _open_event("rift"), 상점/보스/보너스와 충돌 없음.
- 흐름: 선택 → 패널 정리 → 다음 웨이브 진행(스킬 재개).

## 튜닝 여지
균열 +30%·누적 상한, 제단 비용 20·보상 희귀도, 갈림길 +30%/2웨이브·전설 확정. 이벤트 빈도(끝자리). 모두 상수로.
