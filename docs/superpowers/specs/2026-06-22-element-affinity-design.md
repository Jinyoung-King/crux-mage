# 원소 어피니티 빌드 시스템 — 설계 (검증 슬라이스 우선)

**작성일**: 2026-06-22
**상태**: 설계 승인 → 구현 계획 대기
**관련**: 원소 정체성 3부작(감전·상태이상 스킬화·속박), `element_empower`(원소 균열), `_card_synergy`(드래프트 큐레이션, 현재 낡음)

---

## 1. 동기 — 세로 성장을 가로 성장으로

**문제(사용자 자가진단):** 현재 게임은 "매 웨이브 성능 좋은 카드 아무거나 뽑아 최대 웨이브 도달 → 코인 모아 모든 능력치 맥스"인 **단조로운 스펙 상승 트레드밀**. 선택에 의미가 없고(아무거나 센 거), 목표가 유한한 노가다라 질림.

**처방:** 성장을 **세로(↑ 숫자)에서 가로(↔ 다른 빌드)로**. 매 런 *다른 아키타입*으로 분기하게 — 안 질리는 로그라이트의 핵심. 이미 만든 **원소 정체성·반응**이 그 가로축이다(지금은 스탯 위 옵션일 뿐, 플레이의 목적이 아님 → 그걸 목적으로).

## 2. 전체 비전 (참고 — 이번 스펙의 구현 대상 아님)

> 마법사 속성이 **앵커**. 한 속성에 투자할수록 **어피니티**가 쌓여 → 스킬이 **눈에 보이게 진화**(FX·위력) → 정점에서 **궁극기로 변신**(전문화 궁극기 / 콤보 궁극기). 전략: **전문화(깊게) vs 콤보(넓게)**, 그리고 **어떤 콤보냐(생 vs 극)**.

### 2.1 오행(五行) 기반 — 콤보의 원리
콤보를 임의의 "두 속성 섞기"가 아니라 **오행 두 순환**에 뿌리내린다(테마·깊이·학습 효과):
- **상생(相生) 콤보 = 증폭·지속** (먹여주는 관계). 순환 `木→火→土→金→水→木`.
  예) 목생화: 목+화 → 화상이 더 세고 길게. 수생목: 수+목 → 속박·장판 강화.
- **상극(相剋) 콤보 = 폭발·격변** (부딪히는 관계). 순환 `木→土→水→火→金→木`. **기존 반응이 이것**: 수극화=증발 ✓, 토극수=빙결파쇄 ✓.
- 적 상성(×1.5/×0.7)은 이미 상극을 씀 — 상생은 이번에 콤보로 처음 활용.

즉 빌드 분기 = **전문화(앵커 깊게)** vs **상생 콤보(부드럽게 증폭)** vs **상극 콤보(격렬하게 폭발)**.
※ 기존 반응이 생/극에 깔끔히 안 맞음(예: 감전=금+수는 금생수=상생인데 폭발형) — 지금 다 안 뜯고, 향후 반응 정리의 렌즈로만 기록.

**이 전체를 한 번에 만들지 않는다.** 핵심 가정("전문화 vs 콤보 선택이 재밌다")이 검증 안 됐기 때문. 시스템이 존재한다고 결정이 재밌어지는 게 아님 — 자동으로 쌓이는 어피니티면 "스펙 트레드밀에 속성 페인트"일 뿐. 그래서:

## 3. 이번 스펙 = 검증 슬라이스

**검증할 단 하나의 질문:** *"내 원소 길을 고르는 게(전문화 vs 콤보) 실제로 설레는, 뻔하지 않은 결정인가?"*

이걸 싸게 검증하려고 아래 4개만 구현한다. **궁극기·5속성 FX 완성·기존 진화(EVOLVE_BRANCHES) 재편은 전부 제외**(검증 후로).

### 3.1 어피니티 (코어, 전 속성 데이터 구동)
- `affinity[element]` = `ANCHOR_BASE`(마법사 속성이면, 예 0.30) + `PER_SKILL`(예 0.20) × 그 속성 장착 스킬 수 + 어피니티 카드 보너스
- **매핑:** `build.element_empower[element] = affinity[element]` → 기존 `eff_power`가 자동으로 그 속성 스킬 위력 ×(1+affinity) (이미 배선됨)
- **반응 강도도 비례:** 그 속성 반응(감전 연쇄 수·증발 폭발·파쇄 추가타 등)을 affinity로 스케일 → 전문화하면 *내 반응이 강해짐*
- **기회비용(핵심):** 어피니티가 *장착 스킬 분산*으로 결정됨. 한 속성 몰빵 = 높은 단일 어피니티(전문화). 두 속성 = 각자 얕음 + 교차반응 켜짐(콤보). **이 트레이드오프가 검증 대상.**

### 3.2 가시적 진화 (FX, 슬라이스는 1속성부터)
- affinity **티어**(예 ≥0.4 / ≥0.7 / ≥1.0 = T1/T2/T3)마다 그 속성 스킬 FX **단계 진화**(스케일·강도·색). 기존 `pixel_fx` 시트/스케일을 티어로 조정 — **기존 진화 시스템 안 건드리고 *추가* 레이어**(FX 오버레이/스케일)
- **슬라이스 범위:** FX 진화는 **불(fire) 먼저** 구현해 "느낌" 검증(나머지 속성은 어피니티 위력·표시는 받되 티어 FX는 검증 후 확장). 어피니티·반응 스케일·표시는 전 속성.

### 3.3 빌드 정체성 표시 (결정을 읽히게)
- 빌드 화면(또는 HUD)에 **속성별 어피니티 막대** + 현재 아키타입 라벨("🔥 화염 전문화" / "🔥+💧 증발 콤보 ON") + 진화 신호("다음 진화까지 ▮▮▯")
- 드래프트 카드에 **속성 태그 + "이걸 고르면: 화염 어피니티↑" / "물 추가 → 증발 콤보 열림"** 미리보기
- 폰트 미지원 기호(⚔·⟡·⚡ 등) 금지 — 텍스트/지원 기호(▶·★·●)만

### 3.4 드래프트 갈림길 재설계 (낡은 큐레이션 교체)
- `_card_synergy`가 제거된 카드(화염각인·기폭·파쇄)를 참조 → **새 현실(어피니티+오행)으로 재작성**: 내 최대 어피니티 속성(전문화) + 그 속성의 **생/극 콤보 상대** 스킬을 의미 있게 노출(생=증폭, 극=폭발)
- 목표: 드래프트가 "+버프 vs +버프"가 아니라 "**전문화 vs 생콤보 vs 극콤보**"의 진짜 선택

**검증 슬라이스의 구체 갈림길(불 기준):** 화염술사로 시작 →
- **불 전문화** — 화 어피니티 몰빵, 화상·유성 위력·FX 폭발적
- **목+화 (목생화, 증폭)** — 가시밭/가시화살 추가 → 화상 강화·지속(부드러운 증폭)
- **수+화 (수극화, 폭발)** — 빙하/서리 추가 → 증발 버스트(격렬)
→ 셋 다 유효하되 *플레이 리듬이 다르게*. "이 셋 중 뭘 고를까"가 고민되면 검증 성공.

## 4. 제외 — 검증 후로 (의도적)
- **궁극기 변신**(전문화/콤보 캡스톤) — 검증 슬라이스 통과 후 별도 설계
- **5속성 FX 티어 완성** — 슬라이스는 불만
- **기존 진화 재편**(`EVOLVE_BRANCHES`/`_evolve_skill` → 어피니티 구동) — **가장 위험**, 잘 도는 시스템이라 검증 전엔 안 건드림. 슬라이스에선 어피니티 진화 FX가 *추가 레이어*로 공존
- **능동성**(수동성 천장) — 별도 스코프(이번 설계 밖)

## 5. 성공 기준 (검증 판정)
구현 후 **사용자 플레이테스트**로 판정 — 이건 헤드리스 숫자가 아니라 *체감*:
- 드래프트에서 **"전문화 vs 생콤보(증폭) vs 극콤보(폭발)"가 실제로 고민되나?**(셋 다 유효해 보이나, 한쪽이 뻔한 정답인가)
- 어피니티가 오르며 **FX가 진화하는 게 보이고 만족스러운가?**
- "내 빌드가 화염 전문화구나"가 **읽히나?**
- 한 판이 *지난 판과 다르게* 느껴지나?
- → **설레면** 궁극기·확장으로. **안 설레면** 결정 구조(기회비용·밸런스)부터 고침, 큰 투자 전에.

헤드리스로는 "어피니티 값·element_empower 반영·FX 티어 전환·반응 스케일·드래프트 가중"의 *작동*만 검증(체감 아님).

## 6. 제약 / 재사용
- **최소 입력 정체성 유지** — 자동 시전·카드 선택 게임성 그대로(능동성 추가 아님)
- `build.element_empower` + `eff_power` 재사용(어피니티 위력의 토대)
- GPU 파티클·셰이더 금지(웹/모바일), FX는 거버너 tier·MAX_FX 예산 내
- 게임플레이 의도 변경(밸런스) 포함 — 순수 비주얼 아님
- 동작 보존 아님: `_card_synergy` 재작성은 의도적

## 7. 단계 (구현 계획에서 상세화)
1. **어피니티 코어** — affinity 계산 + element_empower 매핑 + 반응 강도 스케일(전 속성). 헤드리스로 값·반영 검증.
2. **빌드 정체성 표시 + 드래프트 갈림길** — 어피니티 막대·아키타입 라벨·카드 미리보기 + `_card_synergy` 재작성. (검증의 핵심 — "결정이 읽히고 고민되나")
3. **가시적 진화 FX(불)** — affinity 티어 → 불 스킬 FX 단계 진화. (검증의 "느낌")
→ 이 3개로 **플레이테스트 → 재미 판정**. 통과 시에만 궁극기·확장.
