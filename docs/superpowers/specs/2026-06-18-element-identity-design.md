# 원소 정체성 시스템 — 상태이상을 카드에서 스킬로 (+ 감전 반응)

**작성일**: 2026-06-18
**상태**: 설계 승인 → 구현 계획 대기
**관련**: `docs/design/2026-06-14-combat-refactor.md`(반응 엔진 Phase 1/2), v3.10 금 비도 재테마(번개=조합 보상 아이디어의 출발)

---

## 1. 동기와 핵심 전환

**문제**: 카드가 스탯 막대기/플래그 토글이라 빌드를 *증폭*만 하고 *분기*시키지 않는다. 특히 상태이상을
카드로 뽑는 구조(화염 각인 = "화상 켜기")는 빌드 정체성을 흐리고 신규 유저에게 복잡도만 더한다.

**전환**: 상태이상·반응을 **카드에서 스킬로 이관**한다.
- 상태이상은 **속성 스킬에 내장된 정체성**이 된다. "무슨 스킬을 끼웠나 = 내 빌드".
- 반응(증발·감전·파쇄·과부하)은 **스킬 조합으로만 창발**한다. 카드 0개로 발동.
- 카드는 순수 강화(공격력·연사·범위 등)와 스킬 획득/진화·행동 변경(관통·다발·장판·처치폭발)에 집중.

이로써 빌드 정체성이 **스킬 로드아웃**에서 나온다 → "매 런이 달라지는" 깊이를 카드 복잡도 증가 없이 얻는다.

---

## 2. 원소 정체성

| 속성 | 정체성 | 역할 | 스킬 내장 효과(기본) |
|---|---|---|---|
| 🔥 화 | **화상(burn)** | 부여 | 모든 화 스킬이 화상 부여 |
| 💧 수 | **둔화(slow)** | 부여 | 모든 수 스킬이 둔화 부여 |
| 🌳 목 | **속박(root)** | 부여(통제) | 목 스킬이 적을 잠시 완전 정지 (⭐ 신규 상태) |
| ⚙️ 금 | **감전(electrocute)** | 반응 | 둔화 적 명중 시 연쇄 번개 |
| 🪨 토 | **파쇄(shatter)** | 반응 | 둔화 적 명중 시 추가타 + 복리관통 |

**창발 조합** (모두 스킬 조합으로만, 카드 없이):
- 화 + 수 = **증발**(물이 화상 적 타격 → 화상 소모 + 광역) · **과부하**(화상+둔화 중첩)
- 금 + 수 = **감전**(연쇄 번개)
- 토 + 수 = **파쇄**(추가타 + 복리관통)
- 목 = **단독 통제**(속박 — 현재 반응 파트너 없음, 향후 확장 여지)

---

## 3. 확장성 원칙 — 스킬 효과는 속성에 하드코딩하지 않는다

**현재**는 스킬 수가 적어 "화=화상, 수=둔화"가 1:1이지만, **스킬 효과는 속성이 아니라 스킬 자신이 선언**한다.
속성 정체성은 *기본 테마/디폴트*일 뿐 스킬이 덮어쓸 수 있다.

**향후 예시**:
- 화 속성 **용암(lava)** = 둔화 부여 (불인데 둔화)
- 수 속성 **동상(frostbite)** = 도트 딜 부여 (물인데 DoT)

**데이터 모델**: `SkillLib.DEFS`의 각 스킬 항목이 자신이 부여하는 상태를 선언한다(예: `applies: "burn"`).
미선언 시 속성 기본값으로 폴백. 반응 트리거(감전·파쇄를 일으키는지)도 동일하게 스킬/속성 단위로 조회 가능한 구조로 둔다.

**구현 방침**: 지금은 속성 기본값만 채우되 **코드를 per-skill 조회 함수로** 작성한다
(예: `SkillLib.applied_status(skill_id)` → 현재는 element 기반 디폴트 반환, 추후 DEFS override 우선).
**동작 변화 0, 확장성 확보** — 용암·동상은 나중에 DEFS 한 줄 + (신규 상태면) StatusEffects 항목으로 추가된다.

## 4. 감전(感電) 반응 — 구체 사양

금속(전도체) 스킬이 둔화(젖은) 적을 명중하면 감전 연쇄가 터진다. v3.10에서 번개를 5속성에서
뺀("번개는 조합으로만 나오는 특별한 것") 후속. 비도술사의 잃어버린 번개가 빌드 조합의 보상으로 돌아온다.

### 트리거
- 위치: `skill_executor._apply_element_reactions(e, element)` — 이미 매 `_skill_hit` 끝에서 호출됨.
- 조건: 공격 속성 == metal AND 대상이 둔화 상태(`e.is_slowed()`).
- **빙결파쇄 재배치**: 현재 `metal/earth + 둔화 = 빙결파쇄`인데, **metal → 감전**, **earth → 빙결파쇄(유지)**로 분리.
  테마도 더 맞음(금속=전기, 바위=파쇄).

### 효과 — 연쇄 번개
- 감전된 적을 기점으로 **가장 가까운 미감전 적 최대 K=3마리**에게 순차 연쇄.
- 연쇄 타격 = 원래 명중 피해(`_hit_ctx.dealt`)의 **0.5배** × 금속 상성 + `take_percent_damage(REACTION_HP_PCT)` 1회
  (증발·파쇄와 동일한 복리관통 → 후반 유효).
- **K 상한 필수** (멀티 인스턴스는 반드시 개수 상한 — 융단폭격 렉 교훈).
- **재귀 차단**: 연쇄는 `_skill_hit`를 거치지 않고 직접 피해 적용(`_explode` 패턴). 무한 감전 방지.

### 비주얼 — 보존 코드 부활, 텍스트 0
- `skill_executor._jagged` + `_lightning_line`(둘 다 "향후 감전 연출용"으로 이미 보존됨)으로 적→적 지그재그 번개.
  연쇄 단계마다 1줄.
- 색 = **전기 푸른-흰**(예 `Color(0.7, 0.85, 1.0)`) — 비도의 강철빛 streak과 구분되어 "감전"을 눈으로 식별.
- **텍스트 팝업 없음**(v2.7에서 폰트 셰이핑이 웹 FPS 최대 적이라 반응명 팝업 제거). 번개 비주얼 자체가 피드백.
- FX 예산: Line2D 수 = K로 제한. `host._hit_spark`는 기존 MAX_FX 게이트가 자동 컷.

### 퍼프
- 감전은 명중 시점 반응이라 발화 빈도가 금속 명중 빈도에 자연 바운드(과부하처럼 상태 재형성 폭주 위험 없음).
  추가 throttle 불필요.

## 5. 속박(root) — 목(木) 신규 상태

목 스킬이 적을 잠시 **완전 정지**시킨다(덩굴/가시로 묶음). 레인 디펜스에서 "기지 도달 차단" 통제 정체성.

- **둔화와 별개 상태**로 구현한다. `slow_factor=0`으로 우회하지 **않는다** — 그러면 목이 묶은 적에
  금속 감전·토 파쇄가 터져 "감전=금+수, 파쇄=토+수" 정체성이 흐려진다. 속박은 목 고유 통제로 격리.
- `StatusEffects`에 `root_time_left` + `is_rooted()` 추가. `apply_move`가 속박 중 속도 0 반환.
- **밸런스 주의**: 평타성 스킬(가시 화살)이 매 발 묶으면 영구 정지 = 과함.
  → **짧은 지속 + 적별 재속박 쿨다운**(과부하의 `OVERLOAD_CD_MS` 벽시계 throttle 패턴 재활용).
- 현재 목 스킬: bolts(가시 화살), thorns(가시밭). 어느 쪽이 어떤 강도로 속박할지는 구현 시 결정
  (예: thorns 장판 안의 적만 속박, bolts는 약하게/쿨다운).
- 반응 파트너는 현재 없음(단독 통제). 향후 "목 반응" 확장 여지로 남김.

## 6. 카드 정리 (5장 제거)

| 카드 | 등급 | 처리 | 이유 |
|---|---|---|---|
| 화염 각인 | common | **삭제** | 상태 부여 — 이제 화 스킬이 직접 줌 |
| 서리 각인 | common | **삭제** | 상태 부여 — 이제 수 스킬이 직접 줌 |
| 기폭 | epic | **삭제** | 반응 증폭 — 위력을 증발 베이스라인에 녹임 |
| 파쇄 | epic | **삭제** | 반응 증폭 — 위력을 파쇄 베이스라인에 녹임 |
| 공명 | legendary | **삭제** | 상태+반응을 한 장에 묶음 — 철학과 정면 충돌 |

**유지**: 메아리(legendary, 상태 무관) + 관통·다발·장판·처치폭발 등 행동 카드 + 모든 강화·스킬 카드.

### 제거에 따른 정리 작업
- `resources/cards/`의 해당 .tres 5개 삭제.
- `main.gd`의 카드 풀 preload 목록에서 참조 제거(누락 시 로드 에러).
- `_is_card_useful` / `_has_burn_source` / `_has_slow_source` 등 드래프트 필터 정리
  (반응 연료원이 카드 → 스킬로 바뀌므로 필터 기준도 스킬 보유로 전환).
- **죽은 모디파이어 처리**: 카드가 사라지면 `build.apply_burn/apply_slow/detonate_burn/frostbite`가
  설정되지 않아 `HitModifierLib`의 `ApplyBurn/ApplySlow/Detonate/Frostbite`가 비활성.
  스킬은 `e.apply_burn/apply_slow`를 **직접** 호출하므로 화/수 스킬 동작은 보존됨.
  미사용 모디파이어 클래스는 보존 또는 제거(구현 시 판단 — 보존 시 주석으로 휴면 표기).
- **반응 베이스라인 강화**: 기폭/파쇄 epic 카드가 주던 증폭분을 보전하기 위해
  증발·파쇄·감전의 기본 수치를 상향 검토(밸런스는 구현·플레이테스트에서 튜닝).

## 7. 발견성 (플레이어가 알게)

- 1차 신호 = 반응 비주얼(증발 광역·감전 푸른 번개·파쇄 추가타).
- `start_screen.gd`의 `TIPS`에 조합 안내 추가:
  - "금속 스킬로 둔화·젖은 적을 치면 감전 연쇄 (금+수 조합)"
  - "스킬마다 고유 효과가 있습니다 — 부여(화상·둔화·속박) 스킬과 반응(감전·파쇄) 스킬을 함께 끼우세요"
- 메모리 교훈: 메커니즘 추가 시 표시·안내를 반드시 동반(안 그러면 "안 먹힌다" 오해).

## 8. 구현 단계 (번아웃 고려 — 각 단계 독립 검증·커밋 가능)

스펙엔 전체 비전을 담되, 구현은 단계로 쪼갠다. 지치면 1단계만 하고 멈춰도 게임은 멀쩡하다.

1. **반응 backbone** (코드 적음, 동작 보존 위주)
   - 감전 추가(metal + 둔화) + 빙결파쇄 흙 전속 분리.
   - 화/수 스킬의 상태 부여를 per-skill 조회 구조로 정리(현재 동작 보존 + 확장 가능).
   - 검증: 카드 없이 금+둔화→감전, 토+둔화→파쇄, 화+수→증발·과부하.

2. **카드 5장 제거 + 풀/필터 정리** (1단계로 스킬 연료 확보 후 안전)
   - .tres 삭제, 풀 preload·드래프트 필터 정리, 죽은 모디파이어 처리, 반응 베이스라인 상향.
   - 검증: 카드풀·드래프트 정상, 화/수 스킬만으로 상태·반응 발생.

3. **목 = 속박 신규 상태 + 밸런스** (가장 새 코드 많음, 독립적이라 마지막)
   - `StatusEffects` root 추가, 목 스킬 적용, 적별 쿨다운, 둔화와 격리.
   - 검증: 목 스킬이 적 정지시키되 영구 정지/렉 없음, 감전·파쇄가 속박엔 안 터짐.

## 9. 검증 기준 (헤드리스, 동작 보존 + 신규)

- 카드 없이 빌드 구성 후: 화 스킬→화상 / 수 스킬→둔화 / 목 스킬→속박 / 금+둔화→감전 연쇄 /
  토+둔화→파쇄 / 화+수→증발·과부하 전부 발생.
- 제거된 카드 5장의 참조(카드 풀·드래프트 필터)가 깨끗 — 로드/드래프트 에러 0.
- 감전: K=3 상한 준수, Line2D 생성 확인, `_skill_hit` 재귀 폭주 없음.
- 속박: 영구 정지/팝업·폭발 폭주(렉) 없음, 감전·파쇄가 속박 상태엔 비발화.
- 헤드리스 부하(고배속·고웨이브)에서 반응 폭주로 인한 엔티티/FX 폭증 없음.

---

## 10. 범위에서 제외 (YAGNI)

- 감전 전용 카드/유물(연쇄 수↑ 등) — 재미 확인 후.
- 마비/스턴(감전) — 이번엔 연쇄만(사용자 선택).
- 감전 스택/상태화 — 명중 시점 반응으로 충분.
- 용암·동상 등 속성 교차 스킬 — 확장성 구조만 갖추고 실제 스킬은 추후.
- 목(木) 반응 파트너 — 단독 통제로 시작, 향후 확장.
