# 이펙트 통일 — 단일 CC0 팩(CodeManu)으로 광역·폭발 FX 일관화

**작성일**: 2026-06-18
**상태**: 설계 승인 대기
**동기**: 사용자 "이펙트가 아쉽다 — 통일성 부족". 진단 결과 *화려함*이 아니라 *스타일 제각각*이 문제.

---

## 1. 문제와 방향

현재 스킬 FX가 **세 출처의 짜깁기**다:
- 불(유성·불바다·흙 폭격) = `fx_ext_explosion.png` (OGA explosion-7, **100×100px**, 컬러)
- 물·목·쇠 = DevWizard "Pixel Art Spells" (`fx_water/wood/metal.png`, **16×32px**, 흑백 틴트)
- 번개(비도·감전) = 절차 생성(`_jagged`+`_lightning_line`)

→ 100px 디테일 폭발과 16px 청키 투사체가 섞여 톤 불일치. **해결 = 한 작가/한 해상도 세트로 광역·폭발을 통일**(코드 갈아엎기 아님 — 아트 소스 통일).

**핵심 원칙(메모리 교훈)**: 화려함↔성능은 상충 아님 — 미리 구운 스프라이트시트는 GPU엔 텍스처 한 장. 단 GPU 파티클·셰이더 금지(웹/모바일 GL 불안정), 동시 FX 수는 거버너 tier·MAX_FX로 제어.

## 2. 채택 에셋

**CodeManu "Free Pixel Effects Pack"** (OpenGameArt)
- **라이선스**: CC0 / 퍼블릭 도메인 (크레딧 불필요, README 명시). 작가: CodeManu / DavitMasia.
- **출처**: https://opengameart.org/content/free-pixel-effects-pack
- **다운로드**: https://opengameart.org/sites/default/files/Free%20Pixel%20Effects%20Pack.zip (1.5MB, 검증 완료)
- **사양**: 100×100px 프레임, 격자 스프라이트시트(예: fire 800×800=8×8=64프레임, freezing 1000×1000=100프레임). 20종, 한 툴(Pixel FX Designer)로 제작 → 스타일 일관.
- **파이프라인 호환**: `pixel_fx.gd`가 이미 격자(`vf` 인자) 지원 — 기존 explosion-7(10×5)과 동일 방식. **재생기 변경 0**.
- **provenance**: 외부 에셋 커밋 규칙 준수 — CC0 확인·출처를 에셋 폴더 주석/커밋/CHANGELOG에 기록.

## 3. 속성 → 이펙트 매핑 (시각 확인 기반)

| 속성 | 스킬(광역/임팩트) | 채택 시트(초안) | 색 |
|---|---|---|---|
| 🔥 화 | 유성·불바다·점화필드 | `11_fire`(화염기둥, 필드), `16_sunburn`/`9_brightfire`(폭발) | 주황-적 ✓ |
| 💧 수 | 빙하·서리바람 | `19_freezing`(얼음 결정) | 청-백 ✓ |
| 🌳 목 | 가시밭 | 녹색 링버스트(`17_felspell` 계열) | 녹색 ✓ |
| 🪨 토 | 융단폭격·낙석 | 적/주황 폭발(`16_sunburn` 등, 불과 다른 변형으로 구분) | 흙빛 주황 △ |
| ⚙️ 금 | 비도 | **절차 강철 streak 유지** + `10_weaponhit` 스파크 | 강철 (유지) |
| ⚡ 감전 | (반응) | **절차 푸른 번개 유지**(`_arc_lightning`) | 푸른-백 (유지) |

- **정확한 프레임 번호·grid는 통합 시 각 시트를 렌더로 확인하며 확정**(초안은 시각 검토로 신뢰성 확보).
- **흙은 근사치**(전용 바위 폭발 없음 → 주황 폭발을 불과 다른 변형으로 차별). 메모리 교훈대로 단일 CC0 세트가 5속성 색을 100% 못 맞춤 — 정직히 근사 매핑.
- **금속·번개는 절차 유지가 정답**: 비도 강철 streak·감전 푸른 번개가 이미 또렷하고 팩에 대응물이 없음. "통일"은 광역·폭발 톤 통일이 목표.

## 4. 범위

**바꾼다**: 광역·폭발·낙하·필드 FX의 스프라이트시트 소스를 CodeManu 팩으로 교체(skill_executor의 `PIXEL_FX.play(...)` 호출들이 참조하는 텍스처 상수).

**안 바꾼다**:
- 비행 투사체(가시 화살 등 16px) — 본질상 작은 투사체라 폭발 팩과 역할 다름. 현행 유지.
- 절차 번개(비도·감전), 데미지 숫자, 적 피격 플래시·스파크.
- 게임플레이·밸런스 일절 무변경(순수 비주얼 교체).

## 5. 구버전 에셋 처리

완전 대체된 에셋은 제거(불필요 번들 축소):
- `assets/sprites/fx_ext_explosion.png`(explosion-7) — CodeManu로 대체 시 제거.
- `assets/sprites/fx_water/wood/metal.png`(DevWizard) — 대응 속성 교체 시 제거(단 metal 스파크는 weaponhit로 대체 or 유지 판단).
- 제거 전 참조(`skill_executor` const preload) 전부 교체 확인 — 누락 시 로드 에러.

## 6. 검증 기준

- 각 속성 스킬 시전 시 새 시트가 올바른 프레임 수·격자로 재생(헤드리스 임시 print + 시각 스폿체크).
- 파싱·로드 에러 0, 제거된 에셋 참조 0.
- 동시 FX 폭주 없음(거버너·MAX_FX 그대로 작동).
- 게임플레이 회귀 0(피해·타겟팅·반응 불변 — FX만 교체).

## 7. 구현 단계 (번아웃 고려, 각 단계 독립 검증·커밋)

1. **에셋 도입 + 파이프라인 확인**: 팩 다운로드→필요 시트만 `assets/sprites/fx_cm_*.png`로 임포트(`--import` 2회), `pixel_fx.gd` 격자 재생 확인. 한 속성(화)만 먼저 연결해 '룩' 확인.
2. **속성별 교체**: 수·목·토를 차례로 CodeManu 시트로 교체(각 속성 = 1커밋, 시각 확인).
3. **정리 + 릴리스**: 구 에셋 제거·참조 정리, 버전 올림·웹 빌드.

## 8. 범위 제외 (YAGNI)

- 캐릭터·몹·배경 아트(이펙트만).
- 금속·번개의 외부 시트화(절차가 더 나음).
- 투사체 100px 확대(역할 다름 — 필요 피드백 시 별도).
- 색 틴트 재색칠(팩이 컬러 → 변형 선택으로 대응).
